TermiForge is a professional SSH, SFTP, and Remote Desktop client built for iPad. Edit remote files as if they were local, tunnel localhost ports through SSH, and stream your Mac's display with full keyboard and trackpad control — over Wi-Fi or the internet.

Built for real remote work

Four connection types, one app. Each one does its job without reinventing the wheel.

SSH Terminal

A real xterm-256color shell with tab completion, ANSI colors, scrollback, and hardware-keyboard support. Sessions stay alive when you switch apps.

SFTP File Manager

Browse, edit, rename, and upload files on any SFTP server. Tap Open With… to edit in Textastic, Runestone, or any editor — saves flow back to the server automatically.

Remote Desktop

Hardware-accelerated H.264 video with Opus audio, sent over DTLS. Full keyboard, trackpad, and game-controller input. Low latency on LAN; adaptive bitrate over the internet.

Web (SSH-tunneled)

Reach localhost:8080, Jupyter, a private admin panel, or any internal web service through an encrypted SSH tunnel. No ports to open, no VPN to configure.

macOS Agent

Install on your Mac in under a minute.

The Mac Agent is a tiny menu-bar app that powers the Remote Desktop feature. It does nothing until the iPad client asks for a session. Open source transport, local-network by default, one drag-and-drop install.

  1. Download the DMG below.
  2. Open it and drag TermiForgeAgent to the Applications folder.
  3. Launch it — it appears in the menu bar. Grant Screen Recording and Accessibility when prompted.
  4. Connect from the TermiForge iPad client.

Privacy isn't a page we wrote.
It's how the product is built.

No analytics. No SDKs.

TermiForge has zero third-party telemetry or tracking. It connects to exactly the servers you configure — and nothing else.

Credentials stay on-device.

SSH keys, passwords, and Mac login details are stored in iOS Keychain with device-only accessibility. They never sync to iCloud and never leave your iPad.

End-to-end encrypted.

SSH for Terminal, SFTP, and Web tunneling. DTLS with a per-session PSK for Remote Desktop streaming. No frames, keystrokes, or files ever pass through our servers — because we don't operate any.
